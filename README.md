# 📐 blueprint.svg

**Layered, editable SVG diagrams from text or images.**

[![License: Apache 2.0](https://img.shields.io/badge/license-Apache%202.0-blue.svg)](LICENSE)
[![Python 3.10+](https://img.shields.io/badge/python-3.10%2B-blue.svg)](https://www.python.org/downloads/)
[![Hugging Face Space](https://img.shields.io/badge/🤗-Space-yellow)](https://huggingface.co/spaces/DanielRegaladoCardoso/blueprint-svg)

LLMs are surprisingly bad at SVG. Ask Claude or GPT for an architecture
diagram and you get hundreds of `<path>` tags glued together — a *blob*
that's impossible to edit, retheme, or version-control meaningfully.

`blueprint.svg` takes a different path: the model never writes SVG.

```
   Text or image
        │
        ▼
   ┌────────────┐
   │    LLM     │   (Qwen 2.5 — text or vision)
   └─────┬──────┘
         │  Diagram IR (JSON)
         ▼
   ┌────────────┐
   │  Layout    │   (Graphviz `dot`/`neato`/`twopi` or custom for sequence)
   └─────┬──────┘
         │  positions
         ▼
   ┌────────────┐
   │  Renderer  │   (deterministic Python → SVG)
   └─────┬──────┘
         ▼
     SVG output with
       • <g id="auth-service">      ← semantic ids, not "path-127"
       • var(--bp-primary)          ← retheme by overriding CSS vars
       • <title>/<desc>             ← ARIA accessible
       • IR JSON saved alongside    ← re-render, diff in git
```

## What you get vs. raw LLM SVG

| | Claude / GPT raw SVG | `blueprint.svg` |
|---|---|---|
| Per-node addressable | ❌ blob of paths | ✅ `<g id="api-gateway">` |
| Theming | rewrite the SVG | ✅ override 3 CSS vars |
| Layout consistency | model guesses positions | ✅ Graphviz solver |
| Round-trip editable | no | ✅ ship the IR JSON |
| Accessibility | none | ✅ `<title>`/`<desc>`/ARIA |

## Quick start

### 1. Hugging Face Space (no install)

Open the [Space](https://huggingface.co/spaces/DanielRegaladoCardoso/blueprint-svg), pick a tab:

- **Text → diagram** — describe what you want, pick the diagram type.
- **Image → diagram** — upload a whiteboard photo / screenshot / sketch and the
  vision model reconstructs it as a clean SVG.
- **Gallery** — render the five bundled example IRs (free, no LLM call).

### 2. Local install

```bash
# system dep — Graphviz `dot` binary
brew install graphviz                # macOS
sudo apt-get install graphviz        # Debian/Ubuntu

# library + app
pip install -e ".[app,dev]"
export HF_TOKEN=hf_...               # for the LLM tabs
python app.py                        # → http://localhost:7860
```

### 3. Library use (no UI, no LLM)

If you already have an IR (hand-written, or generated by your own pipeline),
the renderer works standalone:

```python
import json
from blueprint_svg import Diagram, layout_diagram, render_svg

ir = json.load(open("examples/architecture_web_app.json"))
diagram = Diagram.model_validate(ir)
layout  = layout_diagram(diagram)
svg     = render_svg(diagram, layout)

open("out.svg", "w").write(svg)
```

## Supported diagram types

| Type | Layout backend | Best for |
|---|---|---|
| `architecture` | Graphviz `dot` (LR) | service/infra diagrams, AWS-style |
| `flowchart` | Graphviz `dot` (TB) | decision flows, processes |
| `er` | Graphviz `dot` (LR) | database schemas, entity-relationship |
| `sequence` | Custom column layout | request/response, message passing |
| `mindmap` | Graphviz `twopi` | breakdowns, taxonomies |

## Retheming without touching the LLM

Every node carries a *semantic role* (`primary`, `secondary`, `accent`,
`danger`, `muted`, `neutral`) — not a color. The renderer emits CSS
variables, so a brand retheme is three lines:

```css
:root {
  --bp-primary: #ff7a00;
  --bp-secondary: #1d4ed8;
  --bp-accent: #facc15;
}
```

Drop that into any consumer of the SVG and the diagram repaints. No
regeneration needed.

## Configuration

| Env var | Default | What it does |
|---|---|---|
| `HF_TOKEN` | — | Required for the LLM tabs. Set as a Secret in the Space. |
| `BP_TEXT_MODEL` | `Qwen/Qwen2.5-72B-Instruct` | Model used by the *Text* tab. |
| `BP_VISION_MODEL` | `Qwen/Qwen2.5-VL-72B-Instruct` | Model used by the *Image* tab. |

Both models are routed through `huggingface_hub.InferenceClient` with
`provider="auto"`, so any provider on the Inference Providers list works
without code changes.
